## 🎯 Objetivo do Projeto

Este repositório foi criado como um laboratório de estudos para o Módulo 2 do curso de Automação de Testes com Playwright. O foco não é formar um desenvolvedor JavaScript completo, mas sim aprender a linguagem de forma prática e estratégica para construir automações mais confiáveis e fáceis de manter.

- Um QA precisa aprender JavaScript para escrever testes que sejam robustos, legíveis e adaptáveis a mudanças.
- Conhecer a linguagem impacta diretamente a qualidade dos testes automatizados, reduzindo retrabalho e permitindo soluções mais elegantes.
- Compreender lógica de programação antes de criar automações complexas é essencial para evitar scripts frágeis e garantir uma base sólida.

> Um teste automatizado confiável começa com um pensamento lógico claro e um código que seja tão fácil de entender quanto de manter.

## 🧪 Relação com Automação de Testes

| Conceito | Aplicação na Automação |
| --- | --- |
| Funções | Reutilização de ações em testes, redução de duplicação e melhor organização. |
| Arrays | Manipulação de massa de testes, dados de entrada e resultados esperados. |
| Objetos | Representação de elementos da interface, payloads e configurações de testes. |
| Async/Await | Controle de execução dos testes e sincronização com o fluxo do navegador. |
| Condicionais | Validações, fluxos alternativos e tratamento de diferentes comportamentos da aplicação. |
| Promises | Gerenciamento de ações assíncronas como carregamento de páginas e respostas de rede. |
| Tratamento de Erros | Captura de falhas, relatórios precisos e automações mais resilientes. |
| Módulos | Estruturação de testes, páginas e utilitários em arquivos separados e reutilizáveis. |

## 💡 Mentalidade QA

> **"Profissional não confia. Profissional confere."**

Essa mentalidade se aplica aos testes manuais e à automação: é preciso validar cada passo, questionar suposições e garantir que o comportamento da aplicação esteja realmente conforme esperado. Automação não substitui análise; ela amplia a capacidade de verificação do QA.
